# Engineering Philosophy

## 1. Evidence is stronger than confidence

A confident answer without traceable evidence is not engineering knowledge.

## 2. Manuals are not configurations

A manual says what is possible. A configuration says what is intended. Telemetry says what is happening.

## 3. Unknown is a valid and useful state

A system that openly identifies missing evidence is more trustworthy than one that fills gaps with plausible guesses.

## 4. The operator remains responsible

AI assists engineers. It does not replace authorization, judgement, or accountability.

## 5. The safest acquisition method wins

Prefer an existing export over a login. Prefer a read-only report over a screenshot. Prefer a policy document over a guess. Prefer passive observation over active probing.

## 6. Every recommendation needs provenance

A recommendation should be traceable to:
- an artifact;
- a policy;
- a configuration fact;
- an event;
- an engineering rule;
- or an explicitly labeled hypothesis.

## 7. Production systems are not laboratories

No uncontrolled experiments on-air. No hidden write paths. No convenience shortcuts that risk service continuity.

## 8. Knowledge should outlive individuals

A station should not become less understandable because an engineer retires, changes jobs, or is unavailable at 03:00.

## 9. Diagrams must be generated from evidence

A pretty diagram is not useful if it is fictional. Topology should carry confidence and provenance.

## 10. The goal is engineering memory

The project is not a chatbot with manuals. It is a continuously improving memory of how a technical system is built, operated, and understood.
