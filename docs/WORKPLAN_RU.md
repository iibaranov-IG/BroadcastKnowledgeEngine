## Порядок работы

1. Согласовать рабочее название и слоган.
2. Прочитать `01_EXECUTIVE_SUMMARY.md` и внести фактические правки.
3. Сменному инженеру дать `03_PROBLEM_STATEMENT.md` и `07_CODEX_COLLABORATION.md` на проверку реализма.
4. Собрать 3–5 обезличенных кейсов для `examples/`.
5. Добавить одну настоящую, но обезличенную схему в `diagrams/`.
6. Утвердить правила анонимизации.
7. Создать GitHub/GitLab репозиторий только после redaction review.
8. Подготовить короткую landing page / PDF на основе Executive Summary.
9. После появления понятного демо использовать `09_OPENAI_LETTER.md` как основу для обращения.

## Результат первого цикла

Публично безопасный пакет, который показывает:
- проблему;
- архитектуру;
- безопасность;
- распределение ответственности;
- уже работающую инженерную методологию;
- план превращения прототипа в промышленный инструмент.
